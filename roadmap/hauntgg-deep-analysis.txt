================================================================================
  PART 1: WHAT IS HAUNT.GG
================================================================================

haunt.gg is a bio-link / profile page platform in the same space as guns.lol,
carrd.co, linktree, and bento.me. It targets the Discord/gaming/aesthetic
community. Its differentiator is deep customization combined with a clean docs
site, a public API, and a premium tier that is genuinely feature-rich rather
than just cosmetic.

The platform is built around a single public URL per user:
  haunt.gg/username

Users get a profile page they can customize from a dashboard at:
  haunt.gg/dashboard

Documentation lives at:
  help.haunt.gg

The GitHub org (hauntgg) hosts their public docs repo:
  github.com/hauntgg/docs

haunt.gg's tagline: "Create a stunning biolink page that represents your
personal brand. Share all your content in one beautiful, customizable location."

================================================================================
  PART 2: THE FREE TIER — WHAT YOU GET FOR FREE
================================================================================

Free users get:
  - 1 profile (haunt.gg/username)
  - All basic themes (Default, Modern, Simplistic, Portfolio)
  - Basic badge display
  - Up to 15 social/platform links (Default theme)
  - Up to 4 social cards (Modern theme)
  - Basic color customization (icon color, text color)
  - Profile avatar + background image upload
  - View counter (real, server-side)
  - Basic presence (Discord online status)
  - Basic About Me text
  - Profile decoration (frame around avatar)

Free does NOT include:
  - Typewriter descriptions
  - Custom cursor upload
  - Custom fonts / font upload
  - Profile animations
  - Profile enter text/animation
  - Animated borders
  - Click sound effects
  - Custom profile domains
  - Metadata (Discord embed / SEO)
  - Multiple profiles
  - 2-character usernames

================================================================================
  PART 3: PREMIUM TIER — EVERY SINGLE FEATURE
================================================================================

Premium is a one-time lifetime purchase (no subscription).
Price is not listed in docs but referenced as a token/purchase system.

FULL PREMIUM FEATURE LIST:

  1. MULTIPLE PROFILES
     — Up to 5 profiles total on one account
     — Extendable further by spending tokens
     — Each profile has its own URL, settings, customization
     — Multiple accounts also allowed (up to 3 accounts, each must be active
       with actual customizations + links, not blank alts)

  2. 2-CHARACTER USERNAME
     — Normally usernames require 3+ characters
     — Premium unlocks usernames as short as 2 characters
     — Subject to availability

  3. USERNAME COOLDOWN REMOVAL
     — Normally after changing username there is a cooldown period
     — Premium removes this cooldown entirely
     — Change username as many times as you want

  4. CUSTOM PROFILE DOMAINS
     — Point any domain you own to your haunt.gg profile
     — e.g., yourname.com shows your haunt.gg page
     — Full DNS-level custom domain support

  5. PROFILE METADATA
     — Discord Embed customization:
         * Embed title (what shows in Discord link preview)
         * Embed description
         * Embed color (the left-side accent bar color in Discord)
         * Embed image
     — Tab Title: what appears in the browser tab
     — Favicon: custom icon that appears in browser tab
     — This is full OG meta tag / OpenGraph control

  6. TYPEWRITER DESCRIPTIONS
     — Multiple bio messages that cycle with typewriter animation
     — Each message types out, pauses, deletes, next message types
     — Number of messages: unlimited (based on docs)
     — Speed, pause duration: configurable
     — This is the CYCLING multi-message typewriter (not just one static line)

  7. CUSTOM CURSOR
     — Upload any image as a cursor replacement
     — The real OS cursor is hidden, replaced by the image
     — Works identically to the guns.lol open-source clone pattern

  8. FONT SETTINGS
     — Choose from a list of Google Fonts
     — Upload your own custom font file (TTF/OTF)
     — Apply globally or per-element (heading vs body)
     — Font size controls

  9. PROFILE ANIMATION
     — Entrance animation for the entire profile card
     — Options include: Fade, Slide, Bounce, etc.
     — Controls how the card appears when page loads

  10. PROFILE ENTER ANIMATION
      — The "click to enter" start screen animation
      — Custom text for the enter prompt
      — Animation style for the start screen itself

  11. ANIMATED BORDER
      — The profile card border can animate
      — Animation modes: rotating gradient, pulse, glow breathe, shimmer
      — Border color, width, radius all configurable
      — Border animation speed configurable

  12. CLICK SOUND EFFECT
      — A sound plays when certain interactions happen
      — Upload custom audio file for the click sound
      — This is separate from background music
      — Triggers on: page load, button click, or custom trigger

  13. PRIORITY SUPPORT
      — Premium users jump the support queue
      — Direct access to support team

  14. AUDIO MANAGER EXTENDED
      — Upload background music with:
          * Audio file (mp3/ogg)
          * Cover image (album art displayed on profile)
          * Track title
          * Artist name
      — Volume controls
      — Auto-play toggle (with enter gate for browser policy)

  PART 4: THE 4 THEMES — EVERY DETAIL
================================================================================

haunt.gg calls them "Themes" not "Layouts". Accessible at:
  haunt.gg/dashboard/layout

----------------------------------------------------------------------
THEME 1: DEFAULT
----------------------------------------------------------------------

The most feature-rich and customizable theme. Full control over everything.

STRUCTURE:
  - Full-width page
  - Profile banner (wide image at top, like a Twitter/Discord banner)
  - Avatar overlapping the banner (positioned at bottom-left of banner)
  - Username + display name below avatar
  - Badge row below username
  - About Me / bio text block
  - Social links section (up to 15 links)
  - Up to 15 platform cards in a grid/list

UNIQUE TO DEFAULT:
  - Profile Banner image (separate from background)
  - Widest link capacity (15 links)
  - Most customization options available
  - Supports ALL customization features

----------------------------------------------------------------------
THEME 2: MODERN
----------------------------------------------------------------------

Sleek, minimalistic. Focused on social presence rather than content.

STRUCTURE:
  - Centered card layout
  - Avatar (circular, medium size)
  - Username
  - Short bio/status line
  - Up to 4 social cards — each card is a rich integration tile
    showing live data from the platform (not just a link)
  - Social cards chosen from the 20 integrations list

THE 20 SOCIAL CARD INTEGRATIONS (Modern + Portfolio):
  1.  Discord Server    — shows server name, member count, invite button
  2.  GitHub            — shows username, repo count, follower count
  3.  Roblox            — shows display name, friend count
  4.  Telegram          — shows channel/group link
  5.  Last.fm           — shows now playing / recent scrobbles
  6.  Stats.fm          — Spotify stats integration
  7.  Valorant          — shows rank, win rate, agent stats
  8.  Chess             — shows chess.com/lichess rating
  9.  TikTok            — shows follower count, like count
  10. Instagram         — shows follower count
  11. X / Twitter       — shows follower count, tweet count
  12. Steam             — shows games, playtime, status
  13. Spotify           — shows currently playing track
  14. Discord Bot       — shows bot stats (servers, users)
  15. YouTube           — shows subscriber count, video count
  16. Twitch            — shows follower count, live status
  17. Minecraft Server  — shows server status, player count
  18. Clash of Clans    — shows town hall level, trophies
  19. Brawl Stars       — shows brawler count, trophies
  20. Weather           — shows current weather for a location

MODERN LIMIT: 4 social cards max (picks from the 20 above)

----------------------------------------------------------------------
THEME 3: SIMPLISTIC
----------------------------------------------------------------------

The leanest theme. Minimal visual noise. Text-first.

STRUCTURE:
  - Small centered layout
  - Avatar (small, circular)
  - Username (plain, no effects)
  - Bio text
  - Simple icon-only social links (no labels)
  - No cards, no integrations, no banners

CUSTOMIZATION: Very limited by design. Colors only. No effects.
PURPOSE: For users who want something that loads fast and looks clean
         without any flashy elements.

----------------------------------------------------------------------
THEME 4: PORTFOLIO
----------------------------------------------------------------------

The most unique theme. Designed for developers, designers, creators.

STRUCTURE:
  - Wide layout
  - Avatar + name + title (e.g., "Full-Stack Developer")
  - About Me (full Markdown supported)
  - SKILLS SECTION:
      * Skill name
      * Skill level (percentage bar)
      * Skill icon/logo
      * Unlimited skills
  - PROJECTS SECTION:
      * Project title
      * Project description
      * Project tags (tech stack tags)
      * Project link (external URL)
      * Project icon/image
      * Unlimited projects
  - Social cards (same 20 integrations, up to a limit)
  - Footer with view counter

UNIQUE TO PORTFOLIO:
  - Skills with animated progress bars
  - Projects grid with tags
  - Full Markdown in About Me (bold, italic, links, lists, code blocks)
  - Most similar to a developer resume/portfolio site

================================================================================
  PART 5: THE BADGE SYSTEM — EVERY DETAIL
================================================================================

Source: help.haunt.gg/customization/badges

WHAT ARE BADGES:
  Small icons displayed on your profile, earned through account activity,
  purchases, or being granted by haunt.gg staff. They signal status,
  achievements, and membership type.

BADGE DISPLAY SETTINGS (all configurable per user):

  1. BADGE CONTAINER
     — A box/card that wraps all badges
     — Accent color: any hex color for the container background/border
     — Opacity: 0% to 100% (transparent to solid)
     — Border radius: how rounded the corners are
     — Can be disabled (badges float without container)

  2. MONOCHROME MODE
     — Toggle: makes ALL badge icons render in a single color
     — Color picker to choose the monochrome color
     — Useful for matching profile aesthetic (e.g., all badges in white)

  3. GLOW BADGES
     — Toggle: adds a glow/bloom effect around each badge icon
     — Glow color matches badge's own color OR a custom override color
     — Glow intensity: adjustable

  4. INDIVIDUAL BADGE COLOR OVERRIDES
     — Each badge can have its own color set independently
     — Per-badge hex color input
     — Overrides the monochrome setting for that specific badge

  5. BADGE BORDER
     — Toggle: adds a border around each individual badge
     — Border color, width configurable

  6. BADGE POSITION
     — Choose where badges appear: next to username, below username, etc.
     — Order of badges can be dragged to rearrange

BADGE TYPES (examples from docs):
  — Early Supporter badge (for being an early haunt.gg user)
  — Premium badge
  — Staff badge
  — Verified badge
  — Developer badge
  — Partner badge
  — Donor badge
  — Event-specific limited badges

================================================================================
  PART 6: THE LINKS SYSTEM — EVERY DETAIL
================================================================================

Source: help.haunt.gg/customization/links

TWO TYPES OF LINKS:

  TYPE A: PLATFORM LINKS
    — Pick from preset platforms (Discord, GitHub, Twitter, etc.)
    — Each platform has a preset icon
    — Just enter your username or URL
    — Icon auto-populated from platform library

  TYPE B: CUSTOM LINKS
    — Any URL
    — Custom label text
    — Upload custom icon image OR use a URL for the icon
    — Full freedom

LINK MODES:
  — HYPERLINK mode: clicking the link opens the URL (standard behavior)
  — COPYABLE TEXT mode: clicking copies the text to clipboard
      * Perfect for Discord usernames, wallet addresses, phone numbers
      * Shows a copy icon instead of external link icon
      * Confirmation animation when copied

LINK DISPLAY SETTINGS:

  1. ICON COLOR
     — Global: set one color for ALL link icons at once
     — Per-link: override individual link icon color separately
     — Monochrome toggle for all icons

  2. ICON GLOW
     — Toggle glow effect on icons
     — Glow color follows icon color OR custom color
     — Glow strength: adjustable slider

  3. ICON SIZE
     — Slider: 50% to 150% of default size
     — Applied globally to all icons

  4. LINK ORDER
     — Drag and drop to reorder links
     — Order saved to profile instantly

  5. LINK LABEL
     — Show/hide text label next to icon
     — Label text is editable
     — Font size for labels

  6. LINK STYLE
     — Pill shape vs square vs icon-only
     — Background color per link
     — Hover effects

MAX LINKS:
  — Default theme: 15 links
  — Modern theme: 4 social cards
  — Simplistic: fewer (icon-only strip)
  — Portfolio: social links + separate project links

======================================================================
  PART 8: DECORATIONS — FULL DETAIL
======================================================================

Source: help.haunt.gg/customization/decoration

Decorations are cosmetic frames/effects applied around the avatar.

WHAT THEY ARE:
  — Animated or static frames that surround the circular profile picture
  — Like Discord profile decorations or Roblox avatar accessories
  — Each decoration is a separate cosmetic item

HOW THEY WORK:
  — An image/animation layer renders ON TOP of the avatar circle
  — The decoration has a transparent center showing the avatar through it
  — Animated decorations use GIF or CSS/canvas animation

HOW TO GET THEM:
  — Some are earned (early supporter, milestones)
  — Some are purchased with tokens
  — Some are granted by staff
  — Event-limited decorations available during special periods

DECORATION SETTINGS:
  — Enable/disable decoration toggle
  — If multiple decorations owned: pick which one is active
  — Size adjustment (fit to avatar or slightly larger)

WHY THIS MATTERS FOR BRAZY:
  Decorations are a monetization AND engagement mechanic. Users collect them.
  Limited decorations create FOMO. This is the same model as Discord's profile
  decorations, Roblox accessories, or Spotify's profile frames. Users pay for
  cosmetics they can show off. This is extremely powerful for retention.

======================================================================
  PART 10: THE PUBLIC API — EVERY ENDPOINT
======================================================================

Source: help.haunt.gg/endpoint/introduction
Base URL: https://haunt.gg/api (inferred from docs)

The API is public and documented. No authentication required for read
endpoints. Write endpoints (image upload) may require a token.

----------------------------------------------------------------------
LOOKUP ENDPOINTS:
----------------------------------------------------------------------

  GET /lookup/username/{username}
    — Returns full profile data for a username
    — Response includes:
        * uid (unique user ID)
        * username
        * display name
        * avatar URL
        * bio / about me
        * badge list (array of badge objects with id, name, icon URL)
        * links array (platform, URL, label, icon)
        * social cards array (type, data)
        * theme setting
        * premium status
        * view count
        * Discord presence data (if enabled)
        * audio metadata (if set)
        * decoration (if set)
        * customization settings object
    — Use case: embed haunt.gg profiles in other apps, bots, websites

  GET /lookup/uid/{uid}
    — Same as above but by internal user ID instead of username
    — Useful when username changes but UID is permanent

----------------------------------------------------------------------
LEADERBOARD ENDPOINTS:
----------------------------------------------------------------------

  GET /leaderboard/views
    — Returns profiles ranked by total profile view count
    — Paginated
    — Each entry: username, avatar, view count, rank position
    — This is a GLOBAL leaderboard across all haunt.gg users

  GET /leaderboard/badges
    — Returns profiles ranked by number of badges owned
    — Same format as views leaderboard

  GET /leaderboard/uid
    — Returns profiles ranked by UID (oldest accounts first)
    — Shows who has been on haunt.gg longest

----------------------------------------------------------------------
IMAGE HOST ENDPOINTS:
----------------------------------------------------------------------

  POST /imagehost/upload
    — Upload an image to haunt.gg's CDN
    — Returns CDN URL for the uploaded image
    — Used for: link icons, project thumbnails, inline images

  GET /imagehost/folders
    — List all folders in user's image host

  POST /imagehost/folders
    — Create a new folder

  DELETE /imagehost/folders/{folderId}
    — Delete a folder

  GET /imagehost/{folderId}/images
    — List all images in a folder

  DELETE /imagehost/images/{imageId}
    — Delete a specific image

----------------------------------------------------------------------
WHY A PUBLIC API MATTERS:
----------------------------------------------------------------------

Having a public API transforms haunt.gg from a website into a PLATFORM.
Third parties can:
  — Build Discord bots that display haunt.gg profiles
  — Create profile viewers/browsers
  — Build leaderboard sites
  — Integrate haunt.gg data into other tools
  — Create automated profile scrapers/monitors
  — Power "haunt.gg lookup" slash commands in Discord servers

The leaderboard API is especially powerful — it creates competition and
social proof. Users WANT to appear on the views leaderboard. They share
their profiles more to get views. It's a viral loop built into the API.

WHAT BRAZY SHOULD DO:
  Build a public read API from day one. Even just /api/profile/{username}
  returning the same data the profile renders. This enables community
  building around your platform. Discord bots alone would drive massive
  user acquisition.

======================================================================
  PART 12: DESIGN & UX PATTERNS — WHAT MAKES HAUNT.GG FEEL GOOD
======================================================================

These are observations from the product design, not just features:

  1. EVERYTHING IS LIVE PREVIEW
     Every change in the dashboard shows a live preview of the profile
     on the right side of the screen. You don't save and reload — you
     see changes instantly. This is the #1 UX pattern that makes
     customization feel magical.

  2. ORGANIZED SIDEBAR NAVIGATION
     Dashboard left sidebar has clear sections:

  PART 15: MONETIZATION LESSONS FROM HAUNT.GG
======================================================================

haunt.gg's monetization model is smart. Here's what brazy can learn:

  1. ONE-TIME LIFETIME PURCHASE (no subscription)
     This is a HUGE psychological win. Users hate subscriptions.
     A one-time $10-20 feels like infinite value.
     haunt.gg does this. guns.lol does monthly. haunt.gg wins on
     user goodwill even if guns.lol makes more total revenue.
     Consider: brazy Premium = one-time $15 lifetime. Done.

  2. TOKEN ECONOMY FOR EXTENSIONS
     Base premium gives you X profiles. Want more? Buy tokens.
     Tokens can also unlock decorations, special badges, etc.
     This creates ongoing revenue from already-paying users.
     It's not a subscription — it's optional extra purchases.

  3. LIMITED DECORATIONS / EVENT ITEMS
     Time-limited cosmetics create FOMO and urgency.
     "Holiday decoration available for 2 weeks only"
     Users will pay for things they can't get later.
     Discord does this. Roblox does this. It works every time.

  4. FREE TIER THAT MAKES THEM WANT PREMIUM
     The free tier shows premium features as locked/greyed out.
     Users see the animated border option but can't click it.
     They see what they're missing. This drives upgrades.
     Do NOT hide premium features from view — show them, lock them.

  5. SOCIAL PROOF VIA LEADERBOARD
     Top users on the leaderboard have huge profiles with all
     premium features enabled. This is free advertising for premium.
     "How do they have that decoration? Oh I need to get premium."

======================================================================
  PART 16: TECHNICAL ARCHITECTURE OBSERVATIONS
======================================================================

From reading the docs, changelog, and API structure:

  1. LANYARD FOR DISCORD PRESENCE
     haunt.gg uses the Lanyard API (open source project by Phineas)
     Lanyard is a WebSocket + REST service that exposes Discord presence
     Users add the Lanyard Discord bot to a server, it tracks presence
     haunt.gg queries Lanyard's API with the user's Discord ID
     This is the standard way to do Discord presence in bio-link pages
     Brazy should use Lanyard too. It's free and already built.
     GitHub: github.com/Phineas/lanyard

  2. CDN FOR ALL MEDIA
     All user-uploaded assets (avatar, background, audio, cursor, font)
     are served from a CDN, not from the app server directly.
     This is essential for performance. Images served from CDN = fast.
     Use Cloudflare R2, Vercel Blob, or Supabase Storage for this.

  3. LIVE DATA CARDS USE SERVER-SIDE FETCHING
     The 20 social card integrations don't call external APIs from the
     browser. haunt.gg's server fetches the data and caches it.
     This avoids CORS issues, protects API keys, and allows caching.
     A user's GitHub follower count is fetched server-side and cached
     for e.g. 5 minutes. Not live per-request.
     Pattern: edge function / serverless function per integration.

  4. WEBSOCKET FOR REAL-TIME PRESENCE
     Discord presence updates in real time on haunt.gg profiles.
     This requires either WebSocket or Server-Sent Events (SSE).
     Polling every 30 seconds would work too but feels less live.
     Lanyard provides a WebSocket endpoint directly — connect to it
     and push updates to the profile page.

  5. STATIC PROFILE RENDERING
     Profile pages are likely server-rendered (SSR/SSG) for SEO.
     The metadata (OG tags, title, description) needs to be in the HTML
     before JS loads so Discord/Twitter embeds work properly.
     Next.js with generateMetadata() is the standard approach.
     This is probably how brazy works already (Next.js).
