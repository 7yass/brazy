================================================================================
  SECTION 1: THE PAGE ENTRY — START SCREEN
================================================================================

When the page first loads, a full-screen black overlay (#start-screen) covers
everything with z-index: 30 (above everything else).

Inside it, a typewriter animation runs on the text:
  "Click here to see the motion baby"

HOW THE TYPEWRITER WORKS:
  - startIndex starts at 0
  - Every 100ms, one character is added to startTextContent
  - The blinking cursor "|" toggles every 500ms via setInterval
  - Text is just: startText.textContent = startTextContent + (cursorVisible ? '|' : ' ')
  - No library — pure setTimeout recursion

WHY THIS PATTERN EXISTS:
  Browser autoplay policy blocks audio until user interacts. The start screen is
  a trick to get a user click before trying to play music. The moment the user
  clicks the start screen:
    1. startScreen.classList.add('hidden')
    2. backgroundMusic.muted = false
    3. backgroundMusic.play()
    4. Profile block animates in via GSAP
    5. Typewriter for name + bio begins

The same touchstart handler is duplicated for mobile support.

LESSON FOR BRAZY: You use a "Page Enter Text" field (seen in the screenshot —
"who said im black???"). Same concept. The start screen is the unlock gate.
This clone proves the pattern is correct. You're already doing this right.

  This clone hardcodes badges in HTML. In brazy you render them dynamically

  Not strings like "verified".

================================================================================
  SECTION 4: THE PROFILE AVATAR — ORBIT ANIMATION
================================================================================

The avatar is a circular image with an animated orbiting dot around it.

HOW IT WORKS:
  .profile-container has position: relative and a ::after pseudo-element.

  .profile-container::after:
    content: ''
    position: absolute
    top: -2px, left: -2px, right: -2px, bottom: -2px
    border-radius: 50%
    background: radial-gradient(
      circle at 10px 10px,
      var(--primary-color, #00CED1) 10%,
      transparent 15%
    )
    animation: orbit 3s linear infinite
    z-index: -1
    filter: blur(1px)

  @keyframes orbit:
    0%   { transform: rotate(0deg) }
    100% { transform: rotate(360deg) }

The dot is NOT a separate element — it's a radial-gradient painted at position
"10px 10px" inside the pseudo-element, and the entire pseudo-element rotates.
This gives the illusion of a dot orbiting the avatar.

The --primary-color CSS variable changes per theme:
  home-theme:   #00CED1  (teal/cyan)
  hacker-theme: #22C55E  (green)
  rain-theme:   #1E3A8A  (dark blue)
  anime-theme:  #DC2626  (red)
  car-theme:    #EAB308  (yellow/gold)

CLICK BEHAVIOR:
  When user clicks the profile picture, it triggers a "fast orbit":
    profileContainer.classList.remove('fast-orbit', 'orbit')
    void profileContainer.offsetWidth   ← forces reflow to restart animation
    profileContainer.classList.add('fast-orbit')
  
  .profile-container.fast-orbit::after:
    animation: fast-orbit 0.5s linear forwards
  
  After 500ms, it switches back to regular orbit animation.

LESSON FOR BRAZY: The void offsetWidth trick is how you restart a CSS animation
without removing the element from DOM. You force a layout reflow which resets
the animation timeline. This is used in the profile click easter egg. Consider
adding this to brazy's avatar — clicking the avatar does a spin burst.

HOW switchTheme() WORKS — LINE BY LINE:
  function switchTheme(videoSrc, audio, themeClass, overlay, overlayOverProfile)

     This instantly recolors the orbit dot and card border across all elements.

  2. GSAP fade-out the background video:
       gsap.to(backgroundVideo, { opacity: 0, duration: 0.5 })
  
  3. Inside the onComplete callback (after fade-out):
     a. Sets backgroundVideo.src to new video
     b. Pauses old audio, resets to time 0
     c. Sets new audio volume from slider value
     d. Plays new audio
     e. Removes all theme classes from body, adds new one
     f. Hides/shows particle overlays (hacker = matrix, rain = snow)
     g. Adjusts z-index of profile block (overlayOverProfile moves card behind overlay)
     h. Shows/hides the "View Results" button (only visible in hacker theme)
     i. GSAP fade-in the background video: opacity back to 1

  4. After fade-in, restart orbit animation:
       profileContainer.classList.remove('orbit')
       void profileContainer.offsetWidth
       profileContainer.classList.add('orbit')

================================================================================
  SECTION 7: THE 3D TILT EFFECT
================================================================================

This is the mousemove handler on the profile card that tilts it in 3D.

function handleTilt(e, element):
  1. Gets element's bounding rect
  2. Calculates center: centerX = rect.left + rect.width/2
  3. Gets mouse position relative to center: mouseX = clientX - centerX
  4. Maps mouse position to tilt angle (max 15 degrees):
       tiltX = (mouseY / rect.height) * maxTilt
       tiltY = -(mouseX / rect.width) * maxTilt
  5. Applies via GSAP:
       gsap.to(element, {
         rotationX: tiltX,
         rotationY: tiltY,
         duration: 0.3,
         ease: 'power2.out',
         transformPerspective: 1000
       })

On mouseleave: rotationX + rotationY reset to 0 over 0.5s.

This works on BOTH #profile-block AND #skills-block.
Touch support: touchmove fires the same function using e.touches[0] coordinates.

WHY IT LOOKS GOOD:
  transformPerspective: 1000 sets the camera distance. Lower = more extreme tilt.
  The ease: 'power2.out' makes the tilt feel snappy going in, smooth going out.
  The max 15 degree limit prevents it from looking broken on fast mouse moves.

REQUIRES: transform-style: preserve-3d on the element (already set in CSS).

LESSON FOR BRAZY: This effect is ZERO JavaScript overhead beyond GSAP (which you
might already use). The entire effect is ~15 lines. It makes profiles feel alive.
Consider adding it as an optional "Card Effect: Tilt" in your customization panel.
The key CSS requirement is just adding transform-style: preserve-3d to your card.

================================================================================
  SECTION 12: THE TYPEWRITER SYSTEM
================================================================================

There are 3 separate typewriter instances running simultaneously:
  1. Start screen text ("Click here to see the motion baby")
  2. Profile name ("JAQLIV" — types, waits 10 seconds, deletes, retypes)
  3. Bio text (cycles through multiple messages, types + deletes each one)

PROFILE NAME TYPEWRITER:
  const name = "JAQLIV"
  let nameIndex = 0
  let isNameDeleting = false

  function typeWriterName():
    If not deleting and index < length: add one char, index++, setTimeout 300ms
    If deleting and index > 0: remove one char, index--, setTimeout 150ms
    When fully typed (index === length): set isNameDeleting = true, wait 10000ms
    When fully deleted (index === 0): set isNameDeleting = false (retype)

  GLITCH EFFECT: 10% random chance on each step to add class 'glitch' (a CSS
  @keyframes that does tiny translate jitters for 200ms then removes)

  @keyframes glitch:
    0%   { transform: translate(0) }
    20%  { transform: translate(-2px, 2px) }
    40%  { transform: translate(-2px, -2px) }
    60%  { transform: translate(2px, 2px) }
    80%  { transform: translate(2px, -2px) }
    100% { transform: translate(0) }

BIO TYPEWRITER:
  const bioMessages = [
    "Fu*k Guns.lol & Fakecrime.bio got banned too often, so I created my own.",
    '"Hello, World!"'
  ]
  Cycles through bioMessages array. Each message is typed, waits 2000ms, then
  deleted, then next message starts. Typing speed: 150ms/char. Delete: 75ms/char.
  Same 10% glitch chance per character.

CURSOR BLINK:
  Both name and bio have separate setInterval(500ms) that toggles '|' vs ' '
  at the end of the current text. This is independent of the typing speed.

LESSON FOR BRAZY: You already have a Typewriter Description feature. The key
pattern to copy is the MULTI-MESSAGE cycling bio. Instead of one static
typewriter text, you cycle through an array. The delete + retype creates
the illusion that the profile is "thinking" or "updating" in real time.

    (card becomes completely invisible — you see only text/icons floating)
  
  Else:
    profileBlock.style.background = `rgba(0, 0, 0, ${opacity})`
    profileBlock.style.backdropFilter = `blur(${10 * opacity}px)`
    (blur scales with opacity — more transparent = less blur)

IMPORTANT: Even at opacity 0, all interactive elements stay pointer-events: auto
and opacity: 1. The container is invisible but buttons/links still work.

LESSON FOR BRAZY: You have cardOpacity as a configurable setting. This clone
proves the UX of a LIVE slider is very satisfying. Consider adding a live
preview slider in your customization panel that updates the preview in real-time
as the user drags (without saving until they confirm).

================================================================================
  SECTION 14: GSAP — HOW IT'S USED
================================================================================

GSAP (GreenSock Animation Platform) is loaded via CDN:
  <script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.2/gsap.min.js">

All animations in this project use GSAP. Nowhere does the project use plain
CSS transitions for entrance/exit animations. Here's every GSAP call:

  1. Profile block entrance (after start screen click):
     gsap.fromTo(profileBlock,
       { opacity: 0, y: -50 },
       { opacity: 1, y: 0, duration: 1, ease: 'power2.out' }
     )

  2. Background video fade-out (theme switch):
     gsap.to(backgroundVideo, { opacity: 0, duration: 0.5, ease: 'power2.in' })

  3. Background video fade-in (theme switch):
     gsap.to(backgroundVideo, { opacity: 1, duration: 0.5, ease: 'power2.out' })

  4. Profile block slide left out (going to skills):
     gsap.to(profileBlock, { x: -100, opacity: 0, duration: 0.5, ease: 'power2.in' })

  5. Skills block slide in from right:
     gsap.fromTo(skillsBlock, { x: 100, opacity: 0 }, { x: 0, opacity: 1, ... })

  6. Skills block slide right out (going back to profile):
     gsap.to(skillsBlock, { x: 100, opacity: 0, duration: 0.5, ease: 'power2.in' })

  7. Profile block slide in from left (returning):
     gsap.fromTo(profileBlock, { x: -100, opacity: 0 }, { x: 0, opacity: 1, ... })

  8. Skill bars animate to width:
     gsap.to(pythonBar, { width: '87%', duration: 2, ease: 'power2.out' })

  9. 3D tilt on mousemove:
     gsap.to(element, { rotationX, rotationY, duration: 0.3, transformPerspective: 1000 })

  10. Tilt reset on mouseleave:
      gsap.to(element, { rotationX: 0, rotationY: 0, duration: 0.5 })

  11. Theme change reset of profile position:
      gsap.to(profileBlock, { x: 0, opacity: 1, duration: 0.5, ease: 'power2.out' })

EASING CONVENTIONS USED:
  power2.in  — starts slow, accelerates (exit animations)
  power2.out — starts fast, decelerates (entrance animations)
  All durations: 0.3s (fast micro-interactions) to 2s (skill bar fills)

LESSON FOR BRAZY: If brazy uses Framer Motion or CSS transitions instead of
GSAP, the effect is similar but GSAP gives more precise control, especially for
chained animations (fade out video THEN change source THEN fade in). The
onComplete callback pattern is the key — never change video src while visible.

================================================================================
  SECTION 21: KEY PATTERNS TO STEAL (DIRECT RECOMMENDATIONS)
================================================================================

These are concrete things to implement in brazy based on what this clone does:

  A. THE void offsetWidth TRICK
     When you want to restart a CSS animation on an element that's already
     animating, you MUST force a reflow first:
       element.classList.remove('animation-class')
       void element.offsetWidth   ← THIS LINE IS THE SECRET
       element.classList.add('animation-class')
     Without this, removing and re-adding the class does nothing.

  B. GSAP onComplete FOR VIDEO SWAPS
     Never change a video's src while it's visible. Always:
       gsap.to(video, { opacity: 0, onComplete: () => {
         video.src = newSrc
         gsap.to(video, { opacity: 1 })
       }})
     This prevents the black flash when changing video sources.

  C. MULTI-MESSAGE TYPEWRITER
     const messages = ["message one", "message two", "message three"]
     let msgIndex = 0
     When fully deleted: msgIndex = (msgIndex + 1) % messages.length
     Then start typing messages[msgIndex]
     Add this to brazy's typewriter system.

  D. SOCIAL ICON HOVER WITH rotateY
     .social-icon:hover {
       transform: scale(1.4) rotateY(15deg);
     }
     This adds a 3D coin-flip feel to icon hover. Very cheap CSS.

  E. BADGE TOOLTIP PATTERN
     .badge-container { position: relative }
     .badge-container .tooltip { visibility: hidden; opacity: 0; ... }
     .badge-container:hover .tooltip { visibility: visible; opacity: 1 }
     Pure CSS tooltips, no JS needed. Already works for your badge system.

  F. VISITOR COUNT FORMATTING
     visitorCount.textContent = count.toLocaleString()
     Always format view counts with locale commas: 921234 → "921,234"

  G. ORBIT DOT VIA PSEUDO-ELEMENT + CSS VARIABLE
     The rotating accent dot around the avatar is just a ::after pseudo-element
     with a radial-gradient and a rotate animation. Zero extra DOM elements.
     Color is driven by --primary-color which changes per theme/accent choice.

================================================================================
  SECTION 22: BUGS FOUND IN THE OPEN-SOURCE CLONE
================================================================================

  BUG 1: cursorTrailEffect is called but never defined
    In script.js ~line 95: new cursorTrailEffect({ length: 10, size: 8, speed: 0.2 })
    This function does not exist in script.js or index.html.
    It's wrapped in try/catch so it fails silently. Trail effect never works.

  BUG 2: rain_background.mov won't play on many browsers
    Android Chrome and Firefox do not support .mov containers.
    Should be converted to .mp4 or .webm for cross-browser compatibility.

  BUG 3: Visitor count is fake
    Starts at hardcoded 921,234. Every new visitor in a new browser adds 1 to
    their OWN localStorage. Not shared across users. Completely misleading.

  BUG 4: No error handling on video src changes
    backgroundVideo.src = videoSrc sets src directly with no onerror handler.
    If the file is missing, the video silently fails and the background goes black.

  BUG 5: glitch-overlay has no visual content
    The div has CSS (opacity transitions) but no background, no animation, no
    pseudo-element defining what the "glitch" looks like. Effect is invisible.

  BUG 6: Only one font actually renders as intended
    Retro Gaming is loaded from Google Fonts but never assigned to any element.
    It's dead weight on every page load.
