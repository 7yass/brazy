# Brazy.it — ultimate everything file

This is the largest and most detailed master file yet. It is meant to capture everything possible that brazy.it can add, improve, redesign, or build in the future to surpass guns.lol and haunt.gg while keeping the release philosophy free for users instead of putting the best features behind a premium gate.[web:238][web:241][web:242][web:243][web:251][web:254][web:262][web:263][web:264][web:266][web:269][web:275] It includes product ideas, UI systems, engineering architecture, visual standards, interaction patterns, resource references, asset sources, icon guidance, font guidance, sound guidance, layout logic, motion logic, moderation, onboarding, and feature behavior in implementation-level detail.[web:242][web:251][web:254][web:262][web:264][web:269][web:280][web:282][web:283][web:284][web:313][web:315][web:319]

The point is not to add random features. The point is to build the best free profile platform on the market by combining three things: breadth, polish, and taste.[web:242][web:243][web:262] Breadth means having more systems and expressive power than competitors.[web:241][web:251][web:262][web:266] Polish means every important part looks deliberate and premium, especially the public profile and the builder.[web:243][web:254][conversation_history:1] Taste means the product stays sharp, clean, and modern instead of becoming corny, childish, over-glowing, or cluttered.[web:242][page:2]

## Core philosophy

Brazy.it should stand for these ideas:

- The best identity features are free.[web:262][web:266]
- Users should not have to pay to make their page look good.[web:238][web:243][web:262]
- The builder should feel like a creative studio, not a settings spreadsheet.[conversation_history:1]
- The public profile should look premium by default, even before customization.[web:243][web:262]
- Every customization should have a visual preview.[web:237][web:241][conversation_history:1]
- Layouts should be real layout systems, not renamed skins.[web:237][web:269]
- Motion should feel atmospheric and purposeful, not gimmicky.[web:254][web:264]

### Icons — where to get them and how to use them

This was specifically requested, so this section is very explicit.

Main UI icon sources:

- Lucide for a clean modern line style with strong React usage and good breadth.[web:279][web:284][web:287]
- Tabler Icons for a similarly broad, polished open-source line set.[web:284][web:286][web:320]
- Heroicons for a simpler outline/solid set if you want a more minimal core.[web:284][web:321]
- Phosphor if you need multiple weights and a friendlier softer style.[web:279][web:284][web:321]

Brand icon source:

- Simple Icons for platform logos and brand marks such as Discord, Spotify, GitHub, Telegram, YouTube, TikTok, Steam, Roblox, etc.[web:283][web:290][web:319]

Places to search and browse free icon sets:

- Figma Community for free icon packs and UI icon kits.[web:319]
- General open-source icon collection directories like TOOOLS, Untitled UI’s roundup, Speckyboy roundups, and other curated comparison sites for discovery.[web:320][web:321][web:322]
- Cross-library search tools such as IconSearch or other free finders for comparing icons across libraries.[web:283][web:288]

How to use icons correctly:

- Pick one primary UI icon family and stick to it across the product.[web:279][web:284]
- Use brand icons only for brand contexts.[web:283]
- Do not mix stroke widths from many libraries in one screen.
- Keep icon sizes on a small token scale: 14, 16, 18, 20, 24.
- Use icons to support labels, not replace everything.
- Use filled icons sparingly for strong emphasis.

How to make custom icons if needed:

- Start from a simple 24x24 grid.
- Match stroke width and corner radius to your main icon family.
- Keep paths clean and minimal.
- Test at 16px and 20px, because that is where ugly icons fail.
- Export as optimized SVG.
- Add React wrappers that inherit color and size.
