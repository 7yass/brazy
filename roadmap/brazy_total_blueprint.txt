# Brazy.it — total product blueprint to surpass guns.lol and haunt.gg

This document is the expanded master blueprint for making brazy.it better than both guns.lol and haunt.gg, while keeping everything free at launch and removing the artificial premium gate around core customization, themes, effects, layouts, fonts, cursors, sounds, templates, and profile expression.[web:238][web:241][web:242][web:243][web:251][web:254][web:262][page:1][page:2] The product target is not just "a bio link site" but a full identity platform, live profile builder, theme system, modular content system, public microsite engine, and creator-status layer that feels broader, cleaner, and more polished than both competitors.[web:243][web:251][web:254][web:262][page:2]

The strategy is simple: take every capability that currently creates perceived status on guns.lol and haunt.gg, remove the paywall, improve the implementation quality, improve the visual taste, make the public profile page more beautiful by default, and give users more control without making the builder ugly or overwhelming.[web:241][web:242][web:254][web:262][web:264][page:1] That means better systems, better defaults, better motion, better templates, better layout families, better widgets, better onboarding, better performance, better mobile behavior, and a much stronger actual profile page than either platform currently signals publicly.[web:242][web:243][web:251][web:262][web:269][page:2]

## Competitive scope

guns.lol publicly emphasizes modern customizable bio pages, templates, layout settings, a second-tab widget system, profile and cursor effects, and continuous UI improvements such as custom fonts and expanded personalization options.[web:238][web:241][web:242][web:243][web:251][web:254] haunt.gg publicly advertises that premium includes everything in free plus more profiles, animations, custom fonts, cursors and sound effects, custom profile domains, decorations, themes, templates, assets, and other ongoing additions in its changelog and docs.[web:262][web:263][web:264][web:266][web:269][page:1][page:2] So the complete free brazy.it target has to include all of those categories and then exceed them in finish, clarity, and composition.[web:242][web:251][web:254][web:262][web:264][web:266]

What you are really competing against is not a checklist. You are competing against a feeling: users want their page to look rare, expensive, personal, responsive, animated, complete, and immediately presentable to friends, communities, and followers.[web:243][web:254][web:262][page:2] If brazy.it gives them more power for free but the defaults look worse or the builder feels uglier, they will still perceive it as behind.[web:242][web:243][web:262]

## Product doctrine

Brazy.it should follow these rules everywhere:

- Every major customization category is free.[web:262][web:266]
- The default profile already looks premium before any customization.[web:238][web:243][web:262]
- Every style choice is previewed visually, never hidden behind ugly raw dropdowns.[web:237][conversation_history:1]
- Layout families are structurally different, not just recolors.[web:237][web:269]
- Motion supports hierarchy and atmosphere, not gimmicks.[web:254][web:264]
- The public profile is the main product surface, not the dashboard.[web:243][web:262]
- Users get expressive power without the product becoming corny or cluttered.[web:242][web:254][page:2]

### 1. Username and identity system

The identity layer should include:

- Username.
- Display name.
- Alternate name or subtitle.
- Tagline or micro-status.
- Pronouns toggleable.
- Location toggleable.
- Age or birthday optional and private by default.
- Timezone optional.
- Discord presence or account badge.[conversation_history:1][web:243]

- Theme-specific badge frame.
- Custom badge row.
- Pinned identity chips like artist, editor, developer, designer, musician, collector.
- Links to external handles.
- Hover tooltips for badges.
- Availability state like open for work, commissions open, requests closed.
- Mood/status color or small aura indicator.

How it should work:

- All identity items are modular and reorderable.
- Every item can be shown or hidden per layout.
- Some identity items can collapse into a meta row on smaller screens.
- Users can choose between compact, balanced, and expanded identity density.
- Identity elements inherit the chosen layout style so they never look pasted in.

How it should be built:

- Store identity in a structured profile schema with per-item visibility and order.
- Render through a layout engine that maps profile schema to layout components.
- Use component tokens for chip radius, border, text scale, spacing, and hover feedback.
- Support optional animation per element with reduced-motion fallbacks.

### 25. Free resource references for building everything

#### Icons

Best free icon libraries:

- Lucide for a clean modern default React icon set with strong TypeScript support.[web:284][web:279]
- Heroicons for clean outline/solid icons, especially if you are already using a Tailwind-heavy stack.[web:284]
- Phosphor for multiple weights and a friendlier more expressive style.[web:284][web:279]
- Tabler Icons for very broad coverage and lots of utility icons.[web:284][web:286]
- Remix Icon for system-style UI symbols and broader variety.[web:284][web:286]
- Simple Icons for brand logos like Discord, Spotify, GitHub, YouTube, Telegram, etc.[web:283][web:290]

Practical recommendation:

- Use one primary UI icon library only, ideally Lucide or Tabler for consistency.[web:279][web:284]
- Use Simple Icons for brand marks only.[web:283][web:290]
- Do not mix five different stroke styles in one interface.

Where to search icons:

- Official library sites.
- Cross-library search tools such as IconSearch or Talos icon finder for fast discovery across open libraries.[web:283][web:288]

#### Fonts

- Google Fonts for safe open-source web fonts.[web:282]
- Fontshare for higher-end free commercial-use families that feel less generic.[web:280][web:282]

Practical recommendation:

- Curate 12–20 good font pairings instead of exposing 300 raw fonts.
- Build preset pairs per layout family.

#### Motion libraries

- Motion for React animation and transitions if you want a strong modern API.
- Lenis for smooth scrolling only if used carefully and not forced everywhere.
- Use native CSS transitions for many microinteractions to keep things light.

#### UI primitives

- Radix UI or similar headless primitives for accessible popovers, dialogs, selects, tooltips, and tabs.
- Shadcn-style component patterns can speed implementation, but all visuals must be reauthored to avoid template feel.

#### Color inspiration

- Build your own token system.
- Use competitor screenshots only as reference, not as a design source of truth.
- Keep theme palettes neutral-first.
